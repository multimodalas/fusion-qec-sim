# v5.6.0 — Spectral Trapping-Set Diagnostics

## 1. Motivation

Trapping sets are small subsets of variable nodes in a Tanner graph
that cause iterative decoders to fail.  When belief propagation (BP)
encounters a trapping set, messages circulate within a localized
subgraph and the decoder converges to an incorrect fixed point or
oscillates indefinitely.

Identifying trapping sets is generally NP-hard for arbitrary codes.
However, spectral analysis of the Tanner graph adjacency matrix
provides a tractable proxy: eigenvectors that concentrate their mass
on small subsets of variable nodes reveal localized graph structures
that may trap BP messages.

The v5.6 diagnostic exploits this spectral-localization connection
to identify candidate trapping-set regions without running BP
decoding, providing a purely graph-theoretic structural diagnostic.

## 2. Spectral Localization and Trapping Sets

The adjacency matrix of a Tanner graph encodes connectivity between
variable and check nodes.  Its eigenvectors describe the fundamental
modes of information propagation across the graph.

A **localized eigenvector** concentrates its amplitude on a small
subset of nodes rather than spreading uniformly.  When such
localization occurs on variable nodes, it indicates a subgraph that
is relatively isolated from the rest of the Tanner graph — precisely
the structural condition that enables trapping behavior.

The connection between spectral localization and decoding failures
has been observed empirically: codes whose Tanner graphs exhibit
highly localized eigenmodes tend to have higher error floors,
suggesting that spectral clusters correspond to pseudocodeword-prone
regions.

## 3. Deterministic Clustering Algorithm

For each spectral eigenvector produced by the v5.4 Tanner spectral
analysis:

1. **Extract variable-node components**: Take the first `n` entries
   of the eigenvector (where `n` is the number of variable nodes),
   discarding check-node components.

2. **Normalize**: Divide the variable-node subvector by its L2 norm
   to obtain unit-magnitude components.

3. **Compute importance**: For each variable node `i`, compute:
   ```
   importance[i] = |v[i]|
   ```

4. **Threshold**: Apply a deterministic threshold:
   ```
   threshold = mean(importance) + std(importance)
   ```
   Nodes with importance exceeding this threshold are considered
   part of the localized spectral cluster.

5. **Record cluster**: Store the indices of above-threshold nodes,
   cluster size, maximum importance, and mean importance.

The algorithm produces a JSON-serializable output:

```json
{
  "cluster_count": 2,
  "clusters": [
    {
      "mode_index": 0,
      "cluster_size": 3,
      "nodes": [0, 1, 4],
      "max_importance": 0.82,
      "mean_importance": 0.71
    }
  ],
  "largest_cluster_size": 3,
  "mean_cluster_size": 2.5
}
```

## 4. Deterministic Design

The diagnostic satisfies all determinism requirements:

- **No randomness**: No sampling, no RNG state, no stochastic operations.
- **Stable iteration order**: Modes are processed in index order;
  cluster nodes are reported in ascending index order via `np.where`.
- **Deterministic threshold**: The mean + std threshold is a pure
  function of the input vector.
- **No hash-dependent ordering**: All operations use explicit array
  indexing.
- **JSON-serializable outputs**: All return values are Python
  primitives (`float`, `int`, `list`, `dict`).
- **No input mutation**: Input arrays are copied before processing.

## 6. Experimental Methodology

To study spectral trapping sets:

1. Select a family of QLDPC codes at varying distances.
2. Run the harness with `--spectral-trapping-sets` across a range of
   physical error rates.
3. For each code instance, the harness records:
   - `cluster_count`: number of localized spectral clusters
   - `largest_cluster_size`: size of the largest cluster
   - `mean_cluster_size`: average cluster size
4. Correlate cluster metrics with FER scaling and other v5 diagnostics.

Key research questions:

- Do codes with larger spectral clusters exhibit higher error floors?
- Do spectral cluster locations overlap with empirically observed
  trapping sets?
- Does cluster size scale with code distance, and if so, how does
  this scaling relate to FER scaling?
- Do spectral trapping-set regions align with the fragile boundary
  directions identified by v5.3 and the localized modes identified
  by v5.4?

## 7. Future Directions

- **Cross-diagnostic correlation**: Correlate spectral cluster
  locations with boundary directions (v5.3), spectral alignment
  (v5.5), and barrier heights (v5.1) to build a unified structural
  fragility profile.
- **Cluster overlap analysis**: Measure overlap between spectral
  clusters from different eigenmodes to identify persistent
  structural weaknesses.
- **Code family comparison**: Compare spectral trapping-set profiles
  across different QLDPC code families to identify construction
  strategies that minimize localization.
- **Threshold sensitivity**: Study how the clustering threshold
  affects diagnostic sensitivity and specificity for predicting
  decoding failures.
