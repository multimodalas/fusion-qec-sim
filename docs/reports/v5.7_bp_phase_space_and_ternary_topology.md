# v5.7.0 — BP Phase-Space Explorer & Ternary Topology Classification

## 1 Introduction

v5.7.0 introduces two new observational diagnostics for the QEC
belief propagation (BP) decoding framework:

1. **BP Phase-Space Explorer** — treats BP decoding as a trajectory
   through an observable phase space and computes deterministic
   trajectory metrics.

2. **Ternary Topology Classifier** — classifies each decoding
   trajectory into one of three topological states in observable
   decoder space:
   - +1 stable success basin
   -  0 boundary / metastable region
   - -1 failure basin

Both modules are purely observational diagnostics that do not modify
decoder internals.  They treat the decoder as a pure function and
produce JSON-serializable, deterministic outputs.

## 2 BP as a dynamical system

Belief propagation decoding can be viewed as a discrete-time dynamical
system.  At each iteration t, the decoder produces an observable
belief vector x(t) that evolves according to the BP update rules.
The trajectory {x(0), x(1), ..., x(T)} traces a path through the
high-dimensional space of belief configurations.

Key dynamical properties of interest:

- **Convergence**: does the trajectory approach a fixed point?
- **Oscillation**: does the trajectory exhibit persistent cycling?
- **Basin structure**: which attractor basin does the trajectory enter?

## 3 Observable phase-space representation

The phase space is constructed from observable decoder states only.
No attempt is made to reconstruct the full hidden directed-edge
message manifold.

The observable state at each iteration is the per-variable belief
vector (LLR history) that the decoder already exposes through its
`llr_history` output.

**Key design choice**: the phase space is an observable embedding of
BP dynamics, not a complete representation of internal message state.

### Residual norms

For a trajectory {x(t)}, the residual norm sequence is:

    residual[t] = ||x(t+1) - x(t)||_2

This measures the magnitude of state change at each iteration and
provides a first-order convergence diagnostic.

### Oscillation score

The oscillation score measures the sign-change rate of first
differences in residual norms:

    delta[t] = residual[t+1] - residual[t]
    oscillation = (number of sign changes in delta) / (length of delta - 1)

A high oscillation score indicates non-monotone convergence behavior,
potentially signaling metastability or limit cycling.

## 4 Spectral projection design

The full belief vector is high-dimensional.  To enable tractable
visualization and analysis, trajectories are projected onto a
reduced coordinate system.

### Spectral basis projection

When Tanner spectral modes are available (from v5.4), the trajectory
is projected onto the leading eigenvectors of the Tanner graph
adjacency matrix, restricted to variable-node components:

    coord[t, k] = dot(x(t), basis[:n, k])

This leverages the existing deterministic spectral decomposition and
provides coordinates that are aligned with the graph's structural
modes.

### Deterministic fallback

When no spectral basis is provided, the diagnostic falls back to the
first min(3, state_dimension) standard basis coordinates.  This
preserves determinism and avoids any stochastic dimensionality
reduction.

**Non-goal**: PCA or other data-dependent projections are explicitly
avoided to maintain determinism across runs.

## 5 Ternary topology classification

The ternary classifier assigns each iteration and the overall
trajectory to one of three states:

### +1 Stable success basin
- Residual norm decays below threshold (converged)
- Syndrome residual is zero (if available)
- Oscillation is low
- Not near a decision boundary

### -1 Failure basin
- Residual norm decays or plateaus (converged)
- Syndrome residual remains non-zero (if available)
- Strong spectral alignment or clear failed convergence

### 0 Boundary / metastable region
- Residual norm does not decay sufficiently
- Oscillation remains significant
- Near a decision boundary
- Classifier cannot confidently assign success or failure

### Evidence integration

The classifier can optionally integrate evidence from existing v5
diagnostics when available:

- **v5.5 alignment** — spectral-boundary alignment score
- **v5.3 boundary** — decision boundary distance
- **v5.1 barrier** — escape barrier magnitude

This provides richer classification when the full diagnostic stack
is enabled, while remaining functional with phase-space data alone.

### Per-iteration trace

The classifier produces a per-iteration ternary trace, enabling
analysis of when and how the decoder transitions between topological
regions during its evolution.

## 6 Deterministic design constraints

Both modules satisfy the QEC determinism requirements:

- No hidden randomness or global RNG state
- No use of Python hash()
- Fixed coordinate ordering and basis-vector sign convention
- No stochastic dimensionality reduction
- All outputs are JSON-serializable with canonical ordering
- Repeated execution produces byte-identical outputs
- No in-place mutation of input arrays
- No decoder-internal modification

## 7 Experimental methodology

### Harness integration

The diagnostics are integrated into the evaluation harness via:

- `--bp-phase-space` — enables phase-space trajectory recording
- `--ternary-topology` — enables ternary classification (implies
  `--bp-phase-space`)

The phase-space explorer uses the per-iteration LLR history already
collected by the harness when iteration diagnostics are enabled.
No additional decoder invocations are required.

### Output format

Results are included in the experiment output JSON:

```json
{
  "bp_phase_space": [
    {
      "trajectory_length": 20,
      "state_dimension": 36,
      "residual_norms": [0.5, 0.3, ...],
      "phase_coordinates": [[1.2, -0.3, 0.1], ...],
      "final_phase_coordinate": [0.8, -0.1, 0.05],
      "oscillation_score": 0.15
    }
  ],
  "ternary_topology": [
    {
      "ternary_trace": [0, 0, 0, 1, 1, 1, ...],
      "final_ternary_state": 1,
      "state_durations": {"positive": 17, "zero": 3, "negative": 0},
      "transition_iteration": 3,
      "classification_reason": "stable success: ...",
      "evidence": { ... }
    }
  ]
}
```

## 8 Limitations and future work

### Current limitations

- The phase space is an observable embedding, not a complete
  representation of BP message-space topology.  The classifier
  operates in observable decoder space and no claim is made to
  perfectly reconstruct hidden message-space topology.

- Per-iteration syndrome residuals are approximated using the final
  syndrome weight, as per-iteration syndrome computation is not
  currently exposed by the decoder.

- The ternary classification thresholds are fixed and may require
  tuning for different code families or noise regimes.

### Future directions

- Integration with per-iteration syndrome weight traces when
  available from the decoder.
- Spectral basis selection strategies for different code families.
- Cross-code topology comparison studies.
- Correlation analysis between phase-space metrics and FER.
