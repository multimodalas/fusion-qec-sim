# v5.5.0 — Spectral–Boundary Alignment Diagnostics

## 1. Introduction

The v5.5 diagnostic measures the geometric alignment between spectral
eigenvectors of the Tanner graph and BP decision boundary directions.
This extends the v5 diagnostic stack by connecting two previously
independent analyses: spectral structure (v5.4) and boundary geometry
(v5.3).

The diagnostic is fully deterministic, observational, and does not
modify decoder internals.

## 2. Motivation

The v5.3 boundary analysis identifies directions in LLR space where
the BP decoder transitions between attractor basins.  The v5.4
spectral analysis identifies localized eigenmodes of the Tanner graph
adjacency matrix.

A natural question arises: do the fragile directions identified by
boundary analysis correspond to the localized spectral modes of the
Tanner graph?  If so, spectral localization may serve as a
structural predictor of decoder fragility — a purely graph-theoretic
signal that predicts dynamical vulnerability.

The v5.5 diagnostic answers this question by computing the cosine
similarity between spectral eigenvectors and boundary directions.

## 3. Algorithm Description

For each spectral eigenvector `v` and a boundary direction `d`:

1. **Sign normalization**: If `v[0] < 0`, flip `v = -v` to enforce
   deterministic orientation.

2. **Vector normalization**: Normalize both vectors to unit length:
   - `v_normalized = v / ||v||`
   - `d_normalized = d / ||d||`

3. **Cosine similarity**: Compute the absolute dot product:
   - `alignment = |v_normalized . d_normalized|`

After processing all modes, the diagnostic reports:

- `alignment_scores`: per-mode alignment values
- `max_alignment`: highest alignment across all modes
- `mean_alignment`: average alignment across all modes
- `dominant_alignment_mode`: index of the mode with highest alignment
- `mode_count`: total number of spectral modes analyzed

## 4. Deterministic Design

The diagnostic satisfies all determinism requirements:

- **No randomness**: No sampling, no RNG state, no stochastic operations.
- **Stable iteration order**: Modes are processed in index order.
- **Deterministic sign convention**: The sign flip rule (`v[0] < 0`)
  ensures consistent orientation across runs.
- **No hash-dependent ordering**: All operations use explicit array
  indexing.
- **JSON-serializable outputs**: All return values are Python primitives
  (`float`, `int`, `list`).
- **No input mutation**: Input arrays are never modified in-place.

## 5. Integration with Existing Diagnostics

The v5.5 diagnostic integrates with the existing v5 stack:

```
v5.1  BP attractor escape barrier analysis
        |
v5.3  BP decision boundary estimation  ───┐
        |                                  |
v5.4  Tanner graph spectral diagnostics ───┤
                                           |
v5.5  Spectral–Boundary Alignment  ◄───────┘
```

In the benchmark harness (`bench/dps_v381_eval.py`), the
`--spectral-boundary-alignment` flag automatically enables:

- `--tanner-spectral-analysis` (v5.4)
- `--bp-boundary-analysis` (v5.3)
- `--bp-barrier-analysis` (v5.1)

The harness extracts spectral eigenvectors from the Tanner graph
adjacency matrix and boundary directions from the boundary analysis
results, then calls `compute_spectral_boundary_alignment()`.

## 6. Experimental Methodology

To study spectral-boundary alignment:

1. Select a family of QLDPC codes at varying distances.
2. Run the harness with `--spectral-boundary-alignment` across a
   range of physical error rates.
3. For each (mode, p, distance) triple, the harness records:
   - `alignment_max`: peak alignment across spectral modes
   - `alignment_mean`: average alignment
   - `dominant_alignment_mode`: which spectral mode aligns most
   - `FER`: frame error rate
   - `boundary_eps`: boundary crossing distance
   - `barrier_eps`: escape barrier magnitude
4. Analyze correlations between alignment metrics and FER scaling.

Key research questions:

- Does high spectral-boundary alignment correlate with higher FER?
- Do the most localized spectral modes (highest IPR) also show the
  highest alignment with boundary directions?
- Does alignment strength scale with code distance?

## 7. Expected Research Outcomes

- **Structural fragility predictor**: If alignment correlates with
  FER, spectral localization becomes a purely graph-theoretic
  predictor of decoder vulnerability.
- **Mode-specific fragility**: Identifying which spectral modes
  contribute to boundary fragility may reveal structural weaknesses
  in specific code families.
- **Design guidance**: Understanding the spectral-boundary connection
  may inform QLDPC code design by identifying graph structures that
  minimize spectral localization near decision boundaries.

## 8. Future Directions

- **Multi-boundary analysis**: Extend to correlate spectral modes
  with multiple boundary directions simultaneously.
- **Spectral filtering**: Use alignment information to construct
  spectral filters that identify the most fragile subspaces.
- **Scaling studies**: Systematic investigation of how alignment
  metrics scale with code parameters (distance, rate, column weight).
- **Cross-family comparison**: Compare alignment profiles across
  different QLDPC code families to identify structural patterns.
